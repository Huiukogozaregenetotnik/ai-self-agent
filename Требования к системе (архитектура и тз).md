# Архитектура и ТЗ: Автоматизация процессов на базе OpenClaw

## 1. Обзор системы
Проект представляет собой распределенную систему ИИ-агентов на платформе **OpenClaw**. OpenClaw устанавливается локально (требуется Node.js >= 22) и использует систему навыков (Skills) для выполнения реальных действий в ОС и браузере. Система разделена на два узла для соблюдения контура корпоративной безопасности.

## 2. Инфраструктура и узлы

### 2.1. Узел 1: Координатор (FirstVDS, Ubuntu)
* **Статус:** Всегда онлайн (24/7).
* **Интерфейс:** Интеграция с Telegram для управления всеми процессами.
* **Роль:** Управление личными процессами (Тренировки, Личный врач). Выступает сервером, принимающим HTTPS-запросы от рабочего Mac через VPN (Pantera/Tailscale).

### 2.2. Узел 2: Исполнитель (Корпоративный Mac M1)
* **Статус:** Работает в фоне, имеет root-доступ и доступ к внутренней сети (Jira, Naumen, Exchange).
* **LLM Роутинг:** Использование LiteLLM для унификации API (переключение между облачным Whisper/OpenAI для R&D и локальными моделями в будущем).
* **Роль:** * Браузерная автоматизация (Playwright + встроенный в OpenClaw Chrome CDP) для рабочих систем и медпорталов.
    * Скрипты для работы с файловой системой (чтение видео, транскрибация).

### 2.3. CI/CD и Хранение знаний (IaC)
* **Пайплайн:** Obsidian (iCloud) -> плагин Obsidian Git -> приватный репозиторий GitHub.
* **Синхронизация:** На Mac и Ubuntu настроены cron-задачи, которые делают `git pull` репозитория. Директория с репозиторием привязана к рабочей директории OpenClaw, обновляя файлы `SKILL.md`.

## 3. Векторы реализации задач

### 3.1. Интерактивная автоматизация медпорталов (Human-in-the-loop)
* **Инструмент:** Навык (Skill) на базе Playwright на Mac.
* **Логика:** Скрипт имитирует вход пациента. При обнаружении поля для ввода проверочного кода, скрипт приостанавливает выполнение (pause) и отправляет вебхук на Координатор (FirstVDS). Координатор пишет в Telegram запрос на код. Получив ответ от пользователя, Координатор передает код на Mac, и скрипт продолжает парсинг, сохраняя PDF-заключения на Google Диск.

### 3.2. Обработка рабочих встреч (3 часа видео/день)
* **Инструмент:** Мониторинг файловой системы на Mac + LiteLLM + Whisper API.
* **Логика:** 1. OpenClaw обнаруживает новое видео в целевой папке.
    2. Извлекает аудио и отправляет на транскрибацию.
    3. LLM формирует саммари и выделяет поручения (Action Items).
    4. Через браузерную сессию Mac создает задачи в Jira и корректирует сроки.
    5. Через доменную авторизацию Exchange рассылает саммари участникам календаря.

### 3.3. Рабочие рутины (Спринт и Согласования)
* **Спринт:** Cron-задача в OpenClaw запускается перед планированием. Скрипт парсит активные доски Jira, формирует текстовый отчет (зависшие задачи, бэклог) и отправляет в Telegram через Координатор.
* **Согласования:** Mac мониторит входящие Exchange. При запросе из телеграма или по почте, запускает сессию Chrome, переходит по ссылке в Naumen и оставляет комментарий от имени авторизованного пользователя.

### 3.4. Личный ИИ-Врач и Тренировки
* **Инструмент:** Навыки на сервере Координатора (Ubuntu).
* **Логика Тренировок:** При пересылке сообщения из чата с парнями, OpenClaw парсит текст, сверяет с историей в Obsidian, запрашивает LLM для корректировки и сохраняет обновленный план тренировки обратно в `.md` файл Obsidian.

## 4. Риски ИБ и ограничения
* **Опасность:** Навыки OpenClaw — это сторонний код, выполняющийся с системными привилегиями.
* **Решение:** Строгое ревью всех `SKILL.md` и скриптов в Obsidian перед коммитом. Запуск OpenClaw на Mac от имени пользователя с ограниченными правами (не root), использование изолированных профилей для браузерной автоматизации.
